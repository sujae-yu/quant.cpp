# Book-in-a-Chat Demo

**"Why do I need 7x longer context?"** This demo answers that question.

It loads the entirety of *Alice in Wonderland* (~26K words, ~34K tokens) into a single prompt and asks the model questions about the book. With standard FP16 KV cache, smaller models run out of memory. With quant.cpp's compressed KV cache (uniform_4b keys + q4 values), the full book fits comfortably.

## Quick Start

```bash
# 1. Build quant.cpp
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(sysctl -n hw.ncpu 2>/dev/null || nproc)

# 2. Download and prepare the book
python3 bench/demo/prepare_book.py

# 3. Run the demo
bash bench/demo/book_chat.sh path/to/model.gguf
```

## What It Does

1. Downloads *Alice in Wonderland* from Project Gutenberg (strips boilerplate)
2. Shows book statistics (words, estimated tokens)
3. Tries loading the book with llama.cpp using FP16 KV cache (expects OOM)
4. Loads the book with quant.cpp using compressed KV (`-k uniform_4b -v q4`)
5. Asks 3 questions about the book and displays the model's answers
6. Prints a comparison summary

## Options

```
bash bench/demo/book_chat.sh <model.gguf> [options]

  --llama-bin <path>   Path to llama.cpp binary (auto-detected if not set)
  --threads <N>        Number of threads (default: 4)
  --ctx <N>            Context size override (default: auto-calculated)
```

## How KV Compression Helps

| Engine     | KV Cache  | Memory per Token | 34K Token Book |
|------------|-----------|------------------|----------------|
| llama.cpp  | FP16      | 256 bytes/tok*   | ~8.5 MB/layer  |
| quant.cpp  | 4-bit K+V | 36 bytes/tok*    | ~1.2 MB/layer  |

*Approximate, varies by model architecture (head dim, num layers).

With a 32-layer model, the difference is ~230 MB vs ~38 MB just for the KV cache -- a 6-7x reduction that determines whether a long-context prompt fits in memory.

## Files

- `prepare_book.py` -- Download and clean Alice in Wonderland (pure Python, no deps)
- `book_chat.sh` -- The demo script (compares llama.cpp vs quant.cpp)
- `alice.txt` -- The cleaned book text (generated by prepare_book.py, not checked in)
